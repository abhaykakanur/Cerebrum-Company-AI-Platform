# Architecture Decision Records (Implementation Phase)

This directory holds ADRs for decisions made **during implementation**
(ADR-021 onward), following the format and governance process defined in
`docs/architecture/specification/09_Governance.md`.

The baseline architectural decisions (ADR-001 through ADR-020 — Modular
Monolith, FastAPI, Next.js, PostgreSQL, Neo4j, Qdrant, Redis, MinIO,
OpenSearch, Hybrid Retrieval, Grounded RAG, Knowledge Graph, Enterprise
Memory, RBAC, Docker Compose, Clean Architecture, Hexagonal Architecture,
Domain-Driven Design, Design System First, Polyglot Persistence) already
exist in
`docs/architecture/specification/107_ADR_Catalog.md` — do not duplicate
them here. This directory is for genuinely new decisions made after CES
Phase 0.

## Format

Every ADR here follows the same structure as the CES catalog: Title,
Status, Context, Decision, Consequences, Alternatives Considered,
Trade-offs, Future Review Date.

## Naming

`ADR-0021-short-title.md`, `ADR-0022-short-title.md`, ... continuing the
numbering from the CES catalog.
